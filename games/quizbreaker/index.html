Quizbreaker

I love the idea of making some sort of digital game that could be played by a group of people in person. This is a very early concept for a trivia game where one user would build a quiz with a number of questions. Then they would invite other players to play. One user would ask the questions and the others would play, either individually or on teams. Since this would be a multiplayer game there are a number of technologies that would have to implemented, along with a number of decisions:

  • How is the quiz stored? Which backend would be used?
  • How is the game synchronized between all the players?
    • How do users access one game
      • Through a code, maybe?
    • What happens when a player drops out
      • What happens if the admin drops out
  • Making the UI in Unity would make the game multiplatform
  • How many questions can the admin build
  • What type of questions?
    • Multichoice, look at an image or video, hear a sound
  • Can the quizzes be saved?
  • Is there a point in making a premium version for this game where the premium version would enable more features?
    • Like saving quizzes, customized avatars, a high score table
